# 閾値伝播（`threshold_contagion`）

> 各ステップで，非アクティブなエージェントは，アクティブな近傍の割合が閾値 θ に達したときのみ活性化します．新たにアクティブになったエージェントは同期的・決定論的なラウンドで活性化されます．
> **フェーズ：** Interaction．**出典：** Granovetter (1978)．**種別：** network contagion（binary state, θ）．

[← Mechanism カタログに戻る](../mechanisms.ja.md)

## 1. 概要

`threshold_contagion` は，汎用の `socsim-mechanisms` クレートにおけるネットワーク伝播ファミリーの
**決定論的閾値**メンバーです．各エージェントは二値の*アクティブ*フラグを持ちます．
1ステップに1回，**同期ラウンド**を実行します．まずステップ開始時にアクティブ集合をスナップショットし，
非アクティブなエージェントは，その近傍のうちアクティブなものの割合が閾値 θ に達したときのみ活性化します．
新たにアクティブになったエージェントは一括で書き込まれるため，ラウンド途中で活性化したエージェントは
次のラウンドまで近傍の閾値にカウントされません．

SI 伝播と異なり，活性化は**決定論的**で，独立したエッジ単位の試行ではなくアクティブな近傍の*割合*に依存します
— 個人は十分な数の接触相手がすでに行動しているときにのみ集合行動に加わるという Granovetter のアイデアを捉えます．
アクティブ集合は単調（非活性化なし）なので，メカニズムは**飽和**時に `request_stop` を呼びます．
新たに活性化するエージェントがいないラウンド，または全員がすでにアクティブなときです．

このメカニズムは**ライブラリ専用**です．`socsim-core` の `BinaryState` および `Neighbors`
能力トレイトを実装する任意のワールド上で動作します．これには**`ModulePack` がありません**
（シナリオ TOML 登録なし）．直接構築して `SimulationBuilder` に追加してください．

## 2. 理論と出典

Granovetter (1978) は**集合行動の閾値モデル**を提案しました．各個人は閾値 — その個人が加わる前に
すでに行動していなければならない他者の割合（または人数） — を持ちます．閾値分布のわずかなずれが
最終的なカスケードのサイズに大きな差を生み，一見似た群衆がまったく異なる結果に至る理由を説明します．

socsim は単一の均質な割合閾値 θ を用います．ステップ開始時に次数 $d_i$ とアクティブ近傍数 $a_i$ を持つ
（いずれもスナップショットから読み取る）非アクティブなエージェント `i` について，`i` は次のときのみ活性化します．

$$\frac{a_i}{\max(d_i, 1)} \;\ge\; \theta$$

ここで $\max(d_i, 1)$ は孤立エージェントのゼロ除算を防ぎます．更新は完全に決定論的で，
乱数は関与しません．実装は `granovetter1973` 再現実装の閾値分岐から移植されています．

## 3. データフロー

![threshold_contagion data flow](../assets/mech-threshold-contagion.svg)

このメカニズムは，アクティブ集合のステップ開始時スナップショットから `is_active(i)` と
`neighbors_of(i)` を読み取り，各非アクティブなエージェントのアクティブ近傍割合を計算し，
θ に達したものを集めて，`set_active(i, true)` で一括書き込みます．他の状態には触れません．

## 4. 6フェーズループにおける位置

エージェントが互いに影響を及ぼし合う **Interaction** フェーズで実行されます．
ここではエッジに沿った社会的閾値超えそのものが相互作用です．

- `apply` 呼び出しの開始時に取得したアクティブ集合のスナップショットを読み取り，
  新たに閾値を超えた各エージェントを単一バッチで活性化します — これによりラウンドは同期的になり，
  スケジューラの活性化順序に依存しません．
- このラウンドで活性化したエージェントは次のラウンドまで近傍の割合に寄与しません．
  スナップショットがラウンド全体のアクティブ集合を固定するため，カスケードは1ステップにつき1波ずつ進みます．
- **飽和**時（新たな活性化なし，または全員アクティブ）には `ctx.request_stop` を呼び，
  `granovetter1973` 再現実装の収束ルールに一致します．

## 5. 状態の読み書きコントラクト

| フィールド | 読み取り | 書き込み | 備考 |
|---|:--:|:--:|---|
| `is_active(i)`（`BinaryState`） | ✓ | ✓ | ステップ開始時にスナップショット；割合が θ に達すると非アクティブからアクティブへ反転． |
| `neighbors_of(i)`（`Neighbors`） | ✓ | | 次数 `d_i` とアクティブ数 `a_i` の両方を（スナップショットに対して）この集合から導出． |

## 6. 依存関係と順序制約

- **上流：** なし．`BinaryState + Neighbors` を実装するワールドのみを必要とします．
  トポロジー（完全グラフ・リング・ネットワーク・格子）は `neighbors_of` を介したワールド側の関心事であり，
  初期シード集合もワールドの責任です．
- **下流：** 不要 — メカニズムは飽和時に `request_stop` で実行を自己終了します．
  アクティブ集合は単調なので，収束ヘルパは不要です．

## 7. パラメータ

| パラメータ | 型 | デフォルト | 意味 |
|---|---|---|---|
| `theta`（θ） | `f64` | `0.5` | 活性化閾値：必要なアクティブ近傍の割合．θ が小さいほどカスケードは起こりやすく広い． |

ModulePack がないため，シナリオ TOML のパラメータブロックもありません．単一フィールドはコンストラクタ引数です．

## 8. 適用方法

このメカニズムは**ライブラリモード専用**です — シナリオ TOML 登録はありません．
`BinaryState + Neighbors` を実装するワールドを用意し，メカニズムを構築して
`SimulationBuilder` に追加します．（ワールドのボイラープレートは
[SI 伝播の例](si-contagion.ja.md#8-適用方法)と同一です．）

```rust
use socsim_mechanisms::ThresholdContagionMechanism;
use socsim_engine::{SequentialScheduler, SimulationBuilder};

// θ = 0.3：近傍の30%がアクティブになったら活性化．
let threshold = ThresholdContagionMechanism::new(0.3);

let mut sim = SimulationBuilder::new(world) // world: BinaryState + Neighbors
    .scheduler(Box::new(SequentialScheduler))
    .seed(42)
    .add_mechanism(threshold)
    .build();
sim.run()?;
```

実行前にワールドで初期アクティブ集合をシードしてください．θ を下げるとカスケードは起こりやすく爆発的になり，
上げると活性化前により広い局所的支持を要求します．

## 9. 決定論性と RNG

**決定論的**です．活性化はスナップショットの割合 `a_i / d_i` と固定閾値 θ のみに依存します．
メカニズムは `ctx.rng` に一切触れないため，実行は初期アクティブ集合から完全に再現可能です．
アクティブ集合は単調なので，実行は常に有限ステップで飽和に到達します．

## 10. 期待される動作

結果はシード集合とネットワークの次数構造に対する θ の相対値によって決まります．

- **小さな θ**（必要な局所的支持が少ない）：初期シードが数波でネットワークを氾濫させる大域カスケードを引き起こせます．
- **大きな θ**：ほとんどのエージェントが必要な割合に達しないため，カスケードはシード付近で停止します
  — θ やシード配置のわずかな変化が，局所封じ込めと全システムカスケードの間でシステムを反転させます（Granovetter のティッピングポイント）．

非活性化がないため，アクティブ割合は非減少であり，実行は固定点（飽和）で終わります．
