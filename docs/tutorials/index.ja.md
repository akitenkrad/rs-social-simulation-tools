# チュートリアル

学習指向の，手を動かして進めるレッスンです．各チュートリアルでは何かを **実際にビルドして実行** し，CIでコンパイルされる実例の本物のコードを解説するため，読んだ内容と実際にコンパイルされる内容が常に一致します．上から順に進めてください．

## 学習パス

```
T0 ──▶ T1 ──┬──▶ T2
            ├──▶ T3   ──▶ T5
            └──▶ T4
```

- まず **T0** でツールに慣れます（Rust不要）．
- 次に **T1** へ — すべてのライブラリ系チュートリアルがこの概念の上に構築されます．
- **T2 / T3 / T4** はT1から枝分かれします．どの順でも構いません（それぞれが1つのクレートファミリを追加で扱います）．
- **T5** ですべてをフルスタックのCLIパスにまとめます．

| チュートリアル | 作るもの | モード |
|---|---|---|
| [T0 — はじめに](00-getting-started.ja.md) | `socsim` CLIでシナリオを実行・スイープ・雛形生成する | CLI，Rust不要 |
| [T1 — 最初のモデル](01-first-model.ja.md) | 収束する「冷却」モデルをゼロから：`WorldState` 1つ，`Mechanism` 1つ | ライブラリ |
| [T2 — ネットワーク上の意見ダイナミクス](02-opinion-network.ja.md) | 有界信頼コンセンサスを `socsim-mechanisms` + `socsim-net` + `socsim-metrics` の再利用で構築 | ライブラリ |
| [T3 — 空間グリッドモデル](03-spatial-grid.ja.md) | 格子上のイベント駆動ボーターCAと空間メトリクス | ライブラリ |
| [T4 — LLM駆動エージェント](04-llm-agent.ja.md) | エージェントがLLMで意思決定する噂モデル，決定論を維持 | ライブラリ + `socsim-llm` |
| [T5 — シナリオパック](05-scenario-pack.ja.md) | メカニズムを `ModulePack` にまとめ，シナリオTOMLから駆動 | フルスタック |

## チュートリアルの位置付け（Diátaxis）

これらのチュートリアルは **学習指向** です：何かをビルドしながら概念を学びます．他のドキュメント種別とは意図的に区別されています — 基本を理解したら，以下を参照してください．

- **ハウツー** — [ユースケース＆レシピ](../usecases.ja.md)：ツールを既に知っている人向けの，タスク指向のランブック（「スイープの実行方法 / 実行の再開 / ポリシーの学習」）．
- **リファレンス** — [CLI](../cli.ja.md)，[ライブラリAPI](../library.ja.md)，[Mechanismカタログ](../mechanisms.ja.md)：すべてのフラグ・トレイト・メカニズムの網羅的な説明．
- **解説** — [設計](../design.ja.md)，[アーキテクチャ](../architecture.ja.md)：*なぜ* — 6フェーズモデル，クレートグラフ，キャリブレーション哲学．

チュートリアルはこれらを繰り返さず，リンクで参照します．ステップで「全フラグはCLIリファレンスを参照」とあれば，そちらを辿ってください．
